# Roadmap

## Phase 0: Project Foundation

- Establish repository rules and documentation structure.
- Confirm the static-site architecture and deployment model.
- Define content model compatibility with the existing Markdown repository.

## Phase 1: Site Architecture

- Decide the Hugo-based site structure.
- Define configuration strategy for navigation, footer links, theme settings, and optional ICP metadata.
- Specify shortcode and rendering support for audio, video, formulas, diagrams, and embedded players.
- Produce the initial architecture note and lock the static-first boundaries.

## Phase 2: Core Theme Implementation

- Define the theme packaging and installation workflow.
- Build the base layout and typography system.
- Implement Rosé Pine Dawn and Rosé Pine themes with system-theme support.
- Build adaptive article cards for thumbnails, excerpts, and metadata.

## Phase 3: Content and Reading Experience

- Implement article pages with rich Markdown rendering.
- Add TOC, reading progress, previous/next navigation, and Waline comments.
- Validate embedded media and iframe handling across common viewport sizes.

## Phase 4: Publishing and Operations

- Finalize build and deploy workflow for Cloudflare Pages and/or GitHub Actions.
- Add validation for broken links, content regressions, and theme behavior.
- Document publishing, preview, and maintenance workflows.

## Phase 5: Extended Documentation

- Expand contributor guidance as the project grows.
- Decide whether a separate wiki is needed based on documentation volume and maintenance cost.
